# Contributing to E2E Chat

First off, thank you for considering contributing to E2E Chat! It's people like you that make the open-source community such a great place to learn, inspire, and create.

### Rules of Engagement

1.  **Security First**: This is an E2EE application. Any changes to the `frontend/src/crypto` directory must be heavily documented and will be subject to rigorous review.
2.  **Test Requirement**: All new features must include unit tests. For frontend, use Vitest. For backend, use Pytest.
3.  **PR Workflow**: Use feature branches and open a Pull Request. CI must pass before merging.

### How to Contribute

1.  **Report Bugs**: Open a GitHub issue with a clear description and steps to reproduce.
2.  **Suggest Features**: Open a GitHub issue to discuss new ideas.
3.  **Submit PRs**:
    - Fork the repository.
    - Create a new branch (`git checkout -b feat/my-new-feature`).
    - Commit your changes with clear messages.
    - Push to your fork and open a Pull Request.

### Development Setup

Refer to the [README](README.md) for local development setup and testing instructions.
